# .codex_task_template.md

## Required Reading

Before making changes, read:

1. `AGENTS.md`
2. `docs/00_PROJECT_INDEX.md`
3. `docs/02_FEATURE_MAP.md`

Only read additional documents when necessary:

- Data structure changes: read `docs/04_DATA_SCHEMA.md`
- Product scope changes: read `docs/01_PRODUCT_SPEC.md`
- Architecture or dependency changes: read `docs/03_ARCHITECTURE.md`
- Task history check: read `docs/05_TASK_LOG.md`
- Execution rule check: read `docs/06_CODEX_RULES.md`

## Task Type

Choose one:

- Project setup
- Feature implementation
- Bug fix
- UI polish
- Data update
- Refactor
- Documentation update

## Target Feature

Describe the specific feature to modify.

Example:

- Country list panel
- Game markers
- Year filter
- Game detail card
- Mock game data

## Target Files

Use `docs/02_FEATURE_MAP.md` to identify the target files.

Do not search the whole project unless the feature is not listed or the mapped files are inconsistent.

## Task Goal

Describe the expected result clearly.

## Constraints

- Modify only files directly related to this task.
- Do not introduce backend/database/auth unless explicitly requested.
- Keep the MVP runnable.
- Prefer simple and stable implementation.
- If creating, deleting, moving, or renaming files, update documentation.
- If changing data fields or TypeScript types, update `docs/04_DATA_SCHEMA.md`.
- If changing feature-file responsibility, update `docs/02_FEATURE_MAP.md`.
- If changing architecture, update `docs/03_ARCHITECTURE.md`.
- After meaningful changes, update `docs/05_TASK_LOG.md`.

## Verification

After implementation, report:

1. files changed
2. implementation summary
3. how to run or verify
4. documentation updated
5. page URL, for example `http://localhost:3000`
6. core page entry file, for example `src/app/page.tsx`
7. core component entry file, for example `src/components/GameEarthApp.tsx`
8. if this task generated UI changes, where the user should see those changes
9. next recommended step
