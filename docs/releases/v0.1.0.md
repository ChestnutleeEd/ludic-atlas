# Ludic Atlas v0.1.0 — Game Earth Initial Release

## Overview

Ludic Atlas / Game Earth is a game cultural atlas for exploring global video games across geography and time. The initial release establishes the core product model: users can move through a 3D globe, inspect country-based game clusters, browse a timeline archive, and review generated game records derived from a local RAWG-based data pipeline.

The MVP is intentionally static at runtime. The browser reads local TypeScript data, local GeoJSON, and cached static cover assets. RAWG ingestion, country inference, and cover caching are handled by local scripts before the app runs.

The release includes four major product foundations:

- 3D Globe exploration by country.
- Game archive and timeline browsing system.
- RAWG dataset integration through local generation scripts.
- Country inference pipeline based on developer / studio country or region.

## Core Features

- 3D Globe rendering
  - Interactive earth view built with `react-globe.gl` and Three.js.
  - Local lightweight world country borders for geographic context.
  - Premium black-gold atlas visual direction for the main Earth Explorer.
  - Overview and Surface camera modes with constrained zoom and drag behavior.
  - Region presets for Global, Europe, East Asia, North America, Latin America, Middle East, South Asia, and Oceania.

- Country-based game clustering
  - Games are grouped by primary developer / studio country or region.
  - Country list panel displays mapped countries, visible game counts, rating summaries, and genre summaries.
  - Selected-country view focuses the globe and filters the right panel to that country's games.
  - Broad region filtering keeps dense global data easier to browse.

- Game markers and interaction system
  - Country aggregate marker mode for lightweight global browsing.
  - Representative game cover marker mode for high-rated and selected-country records.
  - Hover tooltips show title, rating, release year, and genre context.
  - Selected-game detail cards show cover, developer, publisher, platforms, rating, and description.
  - Marker placement uses deterministic country-anchor offsets to reduce overlap and visual jitter.

- Archive / timeline browsing system
  - Game Chronicle / 游戏编年馆 provides a year-based archive for generated game records.
  - Timeline groups games by `releaseYear` and shows counts, cover previews, average rating, top genres, and top platforms.
  - Search, genre filters, platform filters, year-desc sorting, and rating-desc sorting are available.
  - Year Exhibit modal opens a selected-year collection with a dossier-style selected-game panel.
  - Visual direction combines retro archive texture, film-like timeline rhythm, and premium digital museum surfaces.

- Game data pipeline with RAWG API ingestion
  - `scripts/fetch-rawg-games.mjs` generates local static game data.
  - Batch mode uses RAWG `/api/games` pagination with configurable date range, Metacritic range, ordering, page size, and maximum record count.
  - Seed mode supports manually curated records when reliable country assignment is required.
  - Generated records are exported through `src/data/games.generated.ts` and consumed by `src/data/games.ts`.

- Cover image caching system
  - `scripts/cache-rawg-covers.mjs` downloads RAWG cover URLs into `public/covers/rawg/`.
  - Generated `coverImage` values are rewritten to local `/covers/rawg/...` paths when cached.
  - `scripts/compress-rawg-covers.mjs` can convert cached covers to WebP and update generated references.
  - Missing or failed covers fall back to `public/covers/fallback-game-cover.svg`.

## Data System

The data system is built around offline preparation and static runtime consumption.

RAWG fetch pipeline:

- RAWG data is fetched by local Node.js scripts, not by browser components.
- `RAWG_API_KEY` is read from `.env.local` when data generation is needed.
- Batch mode fetches list records from RAWG with configurable date and quality filters.
- Seed mode fetches or searches known records from `scripts/rawg-seeds.mjs` and applies curated country metadata.
- Generated data is written into `src/data/games.generated.ts` as a typed `Game[]`.

Batching strategy:

- Batch mode uses the RAWG `/api/games` list endpoint with paginated requests.
- Defaults include date range, Metacritic range, ordering, page size, and maximum game count.
- Optional recent-game supplement mode can include newer records whose Metacritic fields are incomplete.
- The pipeline filters obvious non-primary content such as DLC, expansions, demos, soundtracks, packs, bundles, and collections.
- Records are deduplicated by RAWG id and normalized title / release-year signals where possible.

Country inference logic:

- The MVP maps each game to one primary country or region.
- The primary rule is developer / studio country or region.
- Seeded records can carry explicit `countryCode` and `countryName`.
- Batch RAWG records initially use `UNKNOWN` / `Global` where developer country is not reliable.
- Reviewed high-confidence inference results can be applied through `scripts/apply-country-inference.mjs`.
- Inference metadata can include `countrySource`, `countryConfidence`, and `countryReason` for reviewability.

Caching mechanism for covers:

- Generated records may start with remote RAWG `background_image` URLs.
- The cover cache script downloads remote images into `public/covers/rawg/`.
- Successful local downloads rewrite `coverImage` to a stable static path.
- Existing non-empty cover files are skipped to avoid unnecessary repeated downloads.
- Failed downloads leave the original URL unchanged so records remain inspectable.
- The UI uses fallback recovery when a cover is missing or fails to load.

Dataset scale:

- The current generated dataset is approximately 1,000 games.
- The repository includes local cached RAWG cover assets for stable visual browsing.
- Some generated records remain globally scoped until country inference is reviewed and applied.

## UI / UX System

Globe interaction model:

- Earth Explorer starts from a product hub and opens into a focused 3D globe workspace.
- Users can rotate, zoom, hover countries, select countries, and inspect game markers.
- The right panel switches between region-scoped country overview and selected-country detail.
- Bottom controls manage year range, marker size, marker mode, camera mode, region, zoom, reset, and rotation.

Camera control system:

- `react-globe.gl` point-of-view control drives the stable Earth Explorer.
- Overview mode provides broader global and regional browsing.
- Surface mode uses closer camera constraints for more immersive country inspection.
- Country focus and region presets use deterministic camera targets.
- Mobile layout collapses the panel during globe interaction so the earth remains visible.

Region / country focus system:

- Region presets filter country lists and game markers without changing source data.
- Selected country state focuses the globe, filters game records, and opens country detail.
- Country marker mode supports low-cost global exploration.
- Game marker mode promotes representative high-rating records while keeping marker volume controlled.

Archive visual system:

- Game Chronicle uses a retro + premium archive direction.
- Timeline cards behave like year records rather than generic game cards.
- Dossier panels separate selected-year summaries from selected-game details.
- Local cover imagery, paper / wood / brass textures, and restrained motion create a cinematic archive surface without requiring runtime media services.

## Known Limitations

- Country coverage is incomplete. Many generated RAWG records still need reviewed country inference.
- Dataset quality depends on RAWG metadata availability and project-level inference review.
- Large datasets may require further marker virtualization, asset loading optimization, and WebGL performance work.
- Some UI elements are still under refinement, especially dense timeline states, long-title handling, and mobile detail density.
- The project does not include a backend database, user accounts, saved collections, payments, or a production recommendation algorithm.
