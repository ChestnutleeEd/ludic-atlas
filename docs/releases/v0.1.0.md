# Ludic Atlas v0.1.0 — 游戏星图初始正式版

## 概览

Ludic Atlas / 游戏星图是一个全球游戏文化图谱，用地理和时间两个维度组织游戏资料。v0.1.0 建立了产品的第一个正式 MVP：用户可以通过 3D 地球探索不同国家 / 地区的代表性游戏，也可以通过游戏编年馆按年份浏览生成的游戏档案。

这个版本的运行模型是静态本地优先：浏览器读取本地 TypeScript 数据、本地 GeoJSON 和本地封面资源。RAWG 抓取、国家 / 地区推断与封面缓存都在本地脚本中完成，不在浏览器运行时请求外部 API。

本次 release 包含四个基础系统：

- 按国家 / 地区探索的 3D 游戏地球。
- 按年份浏览的游戏档案 / 时间线系统。
- 基于 RAWG 的本地数据集成流程。
- 基于开发商 / 工作室国家的国家 / 地区推断流程。

## 下载后如何打开

### 推荐方式：下载系统压缩包

在本 release 的 Assets 区域下载：

- `ludic-atlas-v0.1.0-macos.zip`
- `ludic-atlas-v0.1.0-windows.zip`

解压后：

- macOS：双击 `start-mac.command`
- Windows：双击 `start-windows.bat`

脚本会自动执行：

1. 检查是否安装 Node.js。
2. 如果没有 `node_modules`，执行 `npm install`。
3. 启动 Next.js 本地开发服务器。
4. 尝试打开 `http://localhost:3000`。

如果端口 `3000` 被占用，请以终端输出里的实际地址为准。

### 必要前置条件

当前项目是 Next.js + TypeScript Web 项目，不是原生桌面 App，因此 release 包不是完全免运行时的 `.exe` 或 `.app` 安装包。

首次运行需要：

- Node.js LTS，建议 Node.js 20 或更新版本。
- 可访问 npm registry 的网络环境，用于安装依赖。

Node.js 下载地址：

```text
https://nodejs.org/
```

### macOS 常见问题

如果双击 `start-mac.command` 被系统拦截：

1. 右键点击 `start-mac.command`。
2. 选择“打开”。
3. 再次确认打开。

如果提示没有执行权限，在终端进入解压目录后执行：

```bash
chmod +x start-mac.command
./start-mac.command
```

### Windows 常见问题

如果 Windows SmartScreen 提示未知脚本：

1. 选择“更多信息”。
2. 选择“仍要运行”。

如果双击后窗口一闪而过，请打开 PowerShell，进入解压目录后执行：

```powershell
.\start-windows.bat
```

### 命令行下载与打开

如果使用命令行，不需要下载压缩包：

```bash
git clone https://github.com/ChestnutleeEd/ludic-atlas.git
cd ludic-atlas
npm install
npm run dev
```

打开：

```text
http://localhost:3000
```

如果只想下载 v0.1.0 对应源码：

```bash
git clone --branch v0.1.0 --depth 1 https://github.com/ChestnutleeEd/ludic-atlas.git
cd ludic-atlas
npm install
npm run dev
```

## 核心功能

- 3D 地球渲染
  - 使用 `react-globe.gl` 和 Three.js 构建可交互地球。
  - 使用本地轻量世界国家边界数据。
  - 黑金色文化地图视觉方向。
  - 支持 Overview / Surface 两种相机模式。
  - 支持 Global、Europe、East Asia、North America、Latin America、Middle East、South Asia、Oceania 等区域预设。

- 国家 / 地区游戏聚类
  - 游戏按主要开发商 / 工作室国家或地区归类。
  - 国家列表展示游戏数量、评分摘要和主要类型。
  - 点击国家后聚焦地球并切换到国家详情。
  - 区域筛选让全球数据更容易浏览。

- 游戏标记与交互系统
  - 国家聚合点用于轻量全球浏览。
  - 代表性游戏封面标记用于展示高评分和当前选中国家的游戏。
  - 悬停提示展示标题、评分、年份和类型。
  - 游戏详情卡展示封面、开发商、发行商、平台、评分与简介。
  - 标记位置使用确定性国家锚点偏移，减少重叠和跳动。

- 游戏编年馆 / 时间线浏览
  - 按 `releaseYear` 把游戏组织为年份档案。
  - 年份卡片展示数量、封面预览、平均评分、主要类型和平台。
  - 支持搜索、类型筛选、平台筛选、年份排序和评分排序。
  - 年度展柜 modal 展示年度概览和选中游戏档案。
  - 视觉方向是复古档案馆 + 高级数字展厅。

- RAWG 数据管线
  - `scripts/fetch-rawg-games.mjs` 生成本地静态游戏数据。
  - batch 模式使用 RAWG `/api/games` 分页接口，可配置日期、Metacritic 范围、排序、分页大小和最大数量。
  - seed 模式支持人工维护的高可信国家映射。
  - 生成数据输出到 `src/data/games.generated.ts`，再由 `src/data/games.ts` 提供给前端。

- 封面缓存系统
  - `scripts/cache-rawg-covers.mjs` 把 RAWG 远程封面下载到 `public/covers/rawg/`。
  - 成功缓存后将 `coverImage` 改为本地 `/covers/rawg/...` 路径。
  - `scripts/compress-rawg-covers.mjs` 可把封面压缩为 WebP。
  - 缺失或加载失败时使用 `public/covers/fallback-game-cover.svg`。

## 数据系统

数据系统采用“本地准备、静态运行”的结构。

RAWG 抓取流程：

- 浏览器组件不直接请求 RAWG。
- 需要更新数据时，本地脚本从 `.env.local` 读取 `RAWG_API_KEY`。
- batch 模式按日期和质量条件分页抓取 RAWG 列表。
- seed 模式从 `scripts/rawg-seeds.mjs` 读取人工维护记录。
- 生成结果写入 `src/data/games.generated.ts`。

批处理策略：

- 使用 RAWG `/api/games` 列表接口分页抓取。
- 默认条件包含日期范围、Metacritic 范围、排序、分页大小和最大游戏数。
- 可选 recent supplement 模式，用于纳入 Metacritic 尚不完整的新游戏。
- 过滤 DLC、扩展包、试玩版、原声带、合集、bundle 等非主体内容。
- 尽量按 RAWG id、标准化标题和发行年份去重。

国家 / 地区推断逻辑：

- MVP 中每个游戏只映射到一个主要国家 / 地区。
- 主要规则是开发商 / 工作室所在国家或地区。
- seed 记录可以显式携带 `countryCode` 和 `countryName`。
- batch RAWG 记录在无法可靠判断国家时使用 `UNKNOWN` / `Global`。
- 人工复核后的高置信结果可通过 `scripts/apply-country-inference.mjs` 应用。
- 推断记录可以携带 `countrySource`、`countryConfidence` 和 `countryReason`，便于后续复核。

封面缓存机制：

- 初始生成记录可能包含 RAWG 远程图片 URL。
- 缓存脚本会下载远程封面到 `public/covers/rawg/`。
- 已存在且非空的封面文件会跳过，避免重复下载。
- 下载失败时保留原始 URL，方便继续检查。
- UI 会在图片缺失或加载失败时回退到统一 fallback 封面。

数据规模：

- 当前生成数据约 1000 款游戏。
- release 包包含本地缓存封面资源，保证主要视觉浏览稳定。
- 部分生成记录仍处于 Global / UNKNOWN 状态，需要后续继续做国家推断复核。

## UI / UX 系统

地球交互模型：

- 首页提供地球探索和游戏编年馆两个入口。
- 用户可以旋转、缩放、悬停国家、选择国家和查看游戏标记。
- 右侧面板在区域国家总览和选中国家详情之间切换。
- 底部控制条管理年份范围、标记大小、显示模式、相机模式、区域、缩放、重置和旋转。

相机控制系统：

- 稳定版地球使用 `react-globe.gl` 的 point-of-view 控制。
- Overview 模式用于全球 / 区域浏览。
- Surface 模式用于更近距离的国家观察。
- 区域和国家聚焦使用确定性相机目标。
- 移动端在拖动地球时收起面板，让地球保持可见。

区域 / 国家聚焦系统：

- 区域预设只改变 UI 派生筛选，不改原始国家数据。
- 选中国家会同时驱动地球聚焦、游戏过滤和详情面板。
- 国家聚合模式用于低成本全局浏览。
- 游戏封面模式只提升代表性记录，避免一次性渲染过多封面。

档案视觉系统：

- 游戏编年馆采用复古 + 高级档案馆方向。
- 时间线卡片被设计为年份记录，而不是普通游戏卡。
- 档案面板区分年度概览与单个游戏详情。
- 本地封面、纸张 / 木纹 / 黄铜纹理和克制动效共同构成展厅体验。

## 已知限制

- 国家覆盖仍不完整，许多 RAWG 生成记录还需要人工复核国家 / 地区。
- 数据质量受 RAWG 元数据与本项目推断规则影响。
- 更大数据集需要进一步优化标记渲染、图片加载和 WebGL 性能。
- 部分 UI 仍在打磨，包括密集年份、长标题和移动端信息密度。
- 当前不包含后端数据库、用户账号、收藏系统、支付系统或生产级推荐算法。
